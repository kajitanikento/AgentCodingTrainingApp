## 事前確認

以下を実行して現在の状態を把握する（並行実行）：
- `git status` — 未コミットの変更がないか確認
- `git log --oneline main..HEAD` — このブランチの全コミットを確認
- `git diff main...HEAD` — mainからの全差分を確認

未コミットの変更がある場合はコミットを促す。

## PRボディの作成

`.github/PULL_REQUEST_TEMPLATE.md` の全セクションを埋める。

### PR概要
このブランチの差分から「何をしたか」を1〜2文で記載する。

### 設計判断
新規追加したファイルについて、`docs/ARCHITECTURE.md` と `design-document/` を参照しながら「なぜその場所・構造にしたか」を記載する。

### 重複チェック
実装前に調査したキーワードと結果（既存流用 or 新規作成の理由）を記載する。

### セキュリティチェックリスト
該当する項目にチェックを入れる。「あり」の項目はファイル名・行番号も記載する。

### パフォーマンス考慮
リスト描画・ループ・非同期処理の変更がなければセクションごと削除する。

### UI確認
UIに変更がある場合、変更した画面・確認した状態・スクリーンショットを記載する。
スクリーンショットは `/record` スキルで撮影する。

---

## PR作成

ブランチが remote に push されていない場合は先に push する。

```
gh pr create --title "<タイトル>" --body "$(cat <<'EOF'
<埋めたPRボディ>
EOF
)"
```

作成後、PR の URL をユーザーに報告する。
